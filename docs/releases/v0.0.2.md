# Release Notes v0.0.2

**Release Date:** March 28, 2026

---

## 🎉 What's New

### 🔒 Distributed Locking

Prevent concurrent migrations in multi-instance deployments.

```typescript
const runner = new MigrationRunner(prisma, {
  lock: {
    enabled: true,
    timeout: 30000,
    retryAttempts: 3,
    retryDelay: 1000,
  },
});
```

**CLI Usage:**
```bash
# Fail fast if lock not available
npx prisma-shift run

# Wait for lock (recommended for deployments)
npx prisma-shift run --wait
```

---

### ⏱️ Migration Timeouts

Set custom timeouts per migration with automatic cancellation.

```typescript
const migration: DataMigration = {
  id: "20240325_long_operation",
  name: "long_operation",
  timeout: 300000,  // 5 minutes
  
  async up({ prisma, signal }) {
    if (signal?.aborted) {
      throw new Error("Timeout!");
    }
    // Long-running work...
  },
};
```

### 🪝 Lifecycle Hooks

Run scripts at various migration stages.

```typescript
// prisma-shift.config.ts
export default {
  hooks: {
    beforeAll: "./scripts/backup.ts",
    beforeEach: "./scripts/notify-start.ts",
    afterEach: "./scripts/verify.ts",
    afterAll: "./scripts/cleanup.ts",
  },
};
```

### 📦 Batch Processing Helper

Process large datasets efficiently with automatic pagination.

```typescript
async up({ batch, log }) {
  await batch({
    query: () => prisma.post.findMany({ where: { processed: false } }),
    batchSize: 1000,
    process: async (posts) => {
      await prisma.post.updateMany({
        where: { id: { in: posts.map(p => p.id) } },
        data: { processed: true }
      });
    },
    onProgress: (processed, total) => {
      log(`Processed ${processed}/${total}`);
    },
  });
}
```

### 🎯 Conditional Migrations

Run migrations only when conditions are met.

```typescript
const migration: DataMigration = {
  id: "20240325_feature_x",
  name: "enable_feature_x",
  
  condition: async ({ prisma }) => {
    const config = await prisma.config.findFirst();
    return config?.featureXEnabled === true;
  },
  
  async up({ prisma, log }) {
    log("Feature X is enabled, running migration...");
  },
};
```

### 🔗 Migration Dependencies

Ensure migrations run in the correct order.

```typescript
const migration: DataMigration = {
  id: "20240325_calculate_scores",
  name: "calculate_scores",
  
  requiresSchema: ["20240324000002_add_categories"],
  requiresData: ["20240324010003_setup_profiles"],
  
  async up({ prisma }) {
    // Dependencies are guaranteed to be executed
  },
};
```

### ⏳ Wait Flag (`--wait`)

Wait for lock acquisition instead of failing immediately.

```bash
# Terminal 1
npx prisma-shift run

# Terminal 2 (waits patiently)
npx prisma-shift run --wait
# Waiting to acquire migration lock...
# Another instance is running migrations. Waiting...
# Lock acquired after 4 attempt(s)
```

---

### 📦 With-Schema Flag (`--with-schema`)

Run Prisma schema migrations before data migrations in one command.

```bash
npx prisma-shift run --with-schema
```

Runs:
1. `prisma migrate deploy` - Schema migrations
2. `prisma generate` - Generate Prisma Client
3. `prisma-shift run` - Data migrations

---

### 🔧 Config File Support

Configure via `prisma-shift.config.ts`.

```typescript
import { Config } from "prisma-shift";

export default {
  migrationsDir: "./prisma/data-migrations",
  migrationsTable: "_dataMigration",
  
  logging: {
    level: "info",
    progress: true,
    format: "text",
  },
  
  lock: {
    enabled: true,
    timeout: 30000,
    retryAttempts: 3,
    retryDelay: 1000,
  },
  
  execution: {
    timeout: 0,
    transaction: true,
  },
} satisfies Config;
```

## Improvements

### Transaction Control

Option to disable transactions for long-running migrations.

```typescript
const migration: DataMigration = {
  id: "20240325_long_migration",
  name: "long_migration",
  disableTransaction: true,  // Commits incrementally
  async up({ prisma }) {
    // Each operation commits immediately
  },
};
```

### Better Error Handling

Clear error messages with migration context.

```
✗ Migration failed at: 20240325_migration_name
Error: Could not acquire migration lock. Another instance may be running migrations.
```
